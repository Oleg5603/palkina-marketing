# Мультиагентная структура — проект «Психотерапевт» (Светлана Палкина)

Агенты обработки лидов и продвижения: от первого контакта до записи на сессию.

## Роли

### Координатор (Orchestrator)
Принимает входящее событие (новый лид, сообщение, заявка с лендинга), решает какому
агенту передать задачу, собирает результат. Единственная точка входа в систему.

### Разведчик (Scout)
Ищет потенциальную аудиторию: парсинг тематических групп VK, мониторинг новых
заявок/сообщений. Передаёт сырые контакты/лиды Квалификатору.

### Квалификатор (Qualifier)
Оценивает лида: подходит ли по профилю (отношения, пары, готовность к терапии),
приоритет, нужен ли ручной контакт Светланы. Отсекает нерелевантные обращения.

### Исследователь (Researcher)
Собирает контекст по лиду перед контактом (открытые данные, история переписки,
источник трафика/UTM), готовит краткую справку для Копирайтера/Секретаря.

### Копирайтер (SDR Agent)
Пишет персонализированные сообщения первого контакта и ответы по контент-плану
(VK, Telegram, прогревная цепочка из PLAN.md, этап 7).

### Секретарь (Scheduler)
Ведёт запись на сессии: уточняет удобное время, отправляет напоминания,
эскалирует к Светлане при нестандартных запросах.

### Telegram-агент
Интерфейс в Telegram-боте (см. PLAN.md, этап 5): принимает команды `/start`,
`/price`, `/book`, `/faq`, маршрутизирует сообщения через Координатора.

## Принципы
- Каждый агент — отдельный модуль, без сети завязок на остальных кроме Координатора.
- Секреты (токены ботов, API-ключи) — только через `.env`, не в коде.
